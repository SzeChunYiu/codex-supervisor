# codex-supervisor prompts -- one prompt per non-blank, non-comment line.
# Each prompt spawns one tiled tmux pane and is sent VERBATIM to Codex.
# By default, the supervisor also appends fixed DEBUG and VALIDATOR panes
# unless this file already defines equivalent lanes; set CODEX_SUPERVISOR_DEBUGGER=0
# and CODEX_SUPERVISOR_VALIDATOR=0 only for exceptional one-off runs.
#
# Contract enforced by `codex-supervisor validate-prompts` and `start`:
# - every prompt starts with /goal exactly;
# - every prompt is 50 words or fewer;
# - every prompt references at least one .md file;
# - all details, guardrails, file lists, and stop rules live in markdown.
# For multi-host projects, docs/parallel-sessions.md should import
# docs/distributed-protocol.md and define host/source-tree leases there.
# For dynamic-worker projects, keep generic tasks in codex-tasks/open.txt and
# start N workers with CODEX_SUPERVISOR_DYNAMIC_WORKERS=N.
#
# Recommended pattern:
#   /goal You are PANE <N>, lane <X>. Read docs/parallel-sessions.md and docs/parallel-sessions/<X>.md, then complete one compact-safe iteration.
#
# Anti-pattern: long inline prompts with branch names, file lists, detailed
# cycles, and verification rules. Those grow context and make the TUI typing
# fragile. Move that content to docs/parallel-sessions/<lane>.md instead.

/goal You are PANE 0, lane BUGS. Read docs/parallel-sessions.md and docs/parallel-sessions/lane-template.md, then complete one compact-safe iteration.
/goal You are PANE 1, lane PERF. Read docs/parallel-sessions.md and docs/parallel-sessions/lane-template.md, then complete one compact-safe iteration.
