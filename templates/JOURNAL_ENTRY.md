# Lane journal template

Create one file per lane at `docs/parallel-sessions/journals/<lane>.md`. The
owning lane appends only; other lanes read it and respond through
`meeting_sheet.md` or manager-owned `TEAM_PLAN.md` updates.

## <UTC time> — <lane> — <task/checklist id>
- Read: <files/reports>
- Did: <artifact/commit/report>
- Need from others: <meeting_sheet row or none>
- Lock/lease used: <TEAM_PLAN row>
- Verification: <command/result>
- Next suggestion: <manager queue item or none>

Keep each lane journal under 120 active lines. Archive older entries to
`docs/parallel-sessions/archive/journals/<lane>-<date>.md` and leave a one-line
pointer when the recent context is summarized.
